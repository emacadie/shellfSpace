What is Shellf Space?

This is a site devoted to promoting Linux, BSD and other UNIX-based operating systems for desktop use and in e-commerce. It uses a database to track user-supplied entries about which sites will allow a user to complete a transaction using an open source and/or UNIX-based operating system (such as Linux, FreeBSD, NetBSD, OpenBSD, or Mac OS X) and any of the browsers that run on these systems.

 

If you're doing something on the web that involves 1. logging in/registering and 2. money (either giving some up or checking a balance) and you don't want to use Windows to do it, then add it to the Shellf Space database. And if you try to buy something at a web site and cannot complete the transaction, add that too.

 

So, just like people contribute entries to Linux on Laptops about their experience trying to get Linux to work on a particular laptop, you can contribute entries to Shellf Space about trying to perform web transactions on a UNIX-based system.


What could cause a transaction to be unsuccessful?

Some sites may have JavaScript that only works on IE, or only on Windows. I tried to do my taxes online at two different sites with Mozilla on a Linux machine, and on both sites I couldn't get past a certain point because I couldn't find the button to go to the next page.

 

Some sites may require a Passport/.NET My Services account. These may only work on Windows machines. There are a few efforts to get Passport working on UNIX-based systems, but some people will still refuse to use them. Some will refuse because they do not want to endorse Microsoft technology, some because they don't want all their information in one place, some for both reasons.


What inspired Shellf Space?

Over the past couple of years, there have been some developments that have led people in the free software/open source community to believe that the days of using Linux and other open source OSes to engage in e-commerce may be coming to an end.

 

Some people fear that Microsoft's dot NET (including Passport, Hailstorm, and other technologies) is an attempt to make the Internet into their tollbooth, and prevent other OSes and browsers from being used. One way this could happen is by convincing web sites to require users to have a Passport account to use a site. Given that many sites are already designed to only work with IE, many sites may go along with whatever Microsoft plans for them. (For stories about sites that only work with IE, look here and here.)

 

There have also been a few bills introduced into the United States Congress which would require copy protection technology to be a part of all electronic devices, including PCs. Since copy protection could potentially be circumvented, this would make open source operating systems illegal.

 

As far as I know, there are no sites that track which sites will and will not work with a non-Windows/IE machine. Since I thought there should be one, I decided to create one. Yay me!


How does this site approach Linux/BSD desktop advocacy differently than other desktop advocacy sites?

Generally, when people ask if Linux or BSD is ready for the desktop, they are looking at whether or not these systems are easy to install, and whether or not common and useful applications (such as word processing, spreadsheets and image manipulation applications) exist for these various operating systems.

 

While these are important considerations, they don't address the question of using a UNIX-based system for web transactions. The amount of commerce handled on the web has grown substantially over the past few years, and will continue to do so. As stated above, Microsoft wants to have as much business done on the web on systems using only their software (both operating system and browser). They want to control not only the server side, but the client side as well, through dot NET and DRM (digital rights management). Many software and hardware companies will go along out of fear or laziness. You have probably seen many sites with notices saying they work best (or only) with IE.

 

Think about it like this: what good is a desktop system if you can use it for word processoring or creating spreadsheets, but can't check your bank balance, pay your bills, buy something, or participate in a web forum or message board?


Why does this site spell "shellf" with 2 L's?

Spelling it with 2 L's is a little joke. Although UNIX-based systems can be accessed through a GUI, it is also common to access them via a command-line interface. The commands are interpreted by a program called a shell. Since this site is promoting UNIX-based systems, I thought "Shellf Space" would be a good name.


What do you mean by "UNIX-based system"?

UNIX is a term used to refer to a family of operating systems based on the original UNIX, which was developed at AT&T Labs in the late 60s. It was used as the basis for Sun's Solaris, HP's HP-UX, IBM's AIX and SGI's IRIX.

 

FreeBSD, NetBSD and OpenBSD derived from AT&T's UNIX, although they now contain no AT&T code. They are based on the Berkeley Software Distribution written by University of California-Berkeley's Computing Science Research Group, which evolved into a complete re-write of AT&T UNIX. (There were commercial UNIX systems based on the BSD code, and some, such as Solaris and AIX, are a combination of AT&T and BSD.) Linux is a clone of UNIX, and Apple's OS X is based on FreeBSD.

 

For legal reasons, the companies and individuals who produce some of these operating systems cannot call them "UNIX". However, they are allowed to use the terms "UNIX-like" or "UNIX-based". I just figured typing "UNIX-based system" to refer to PCs running these operating systems would be concise. Check the links page for more about UNIX.

 

The term "UNIX" is a trademark of The Open Group. A good page about The Open Group's relationship to the UNIX standard and UNIX implementation can be found here.


I'm getting some NullPointerException errors. What's the problem?

That seems to happen in a couple of cases.

 

One is if you try to use the site without cookies enabled. The application server used to power this site, Tomcat, uses cookies to track your session. Please enable them to use this site and make the world safe for UNIX-based systems.

 

Another cause is if you try to add a site or update your profile without actually logging in. I have gotten the errors a few times myself, and going to the Login page usually takes care of it. And yes, I am working on it.
